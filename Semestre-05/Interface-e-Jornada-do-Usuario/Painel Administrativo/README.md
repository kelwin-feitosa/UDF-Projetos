## 🎯 Contexto do Produto
Uma empresa de tecnologia está desenvolvendo uma plataforma digital corporativa utilizada por equipes internas para gerenciar usuários, acessos e permissões em diferentes sistemas e serviços. O produto é voltado para uso diário por administradores de sistema responsáveis pela segurança e organização das contas.

## 🚀 Objetivo do Módulo
Permitir que os administradores realizem as seguintes operações com máxima eficiência:
* **Busca:** Localizar rapidamente usuários específicos.
* **Visibilidade:** Visualizar status e permissões atuais.
* **Controle:** Alterar níveis de acesso e gerenciar o ciclo de vida das contas (ativar, suspender ou remover).
* **Segurança:** Executar ações sensíveis com clareza e confirmação.

## 👤 Perfil do Usuário
* **Persona:** Administrador(a) de sistemas experiente.
* **Necessidades:** Alta responsabilidade, pouco tempo disponível e baixa tolerância a erros operacionais.

## 🛤️ Tarefas Detalhadas
O protótipo foca na execução da **Tarefa Principal**: localizar um usuário específico e alterar corretamente seu nível de permissão.

**Tarefas secundárias contempladas:**
- Filtragem de usuários por status ou perfil.
- Visualização de detalhes completos de uma conta.
- Fluxo de confirmação para ações sensíveis.
- Acesso a histórico ou logs básicos de alterações.

## 🛠️ Desafio UX e Diferenciais
A interface foi projetada para resolver os seguintes desafios:
- [x] **Clareza:** Organização de grandes volumes de dados de forma legível.
- [x] **Prevenção de Erros:** Design focado em evitar cliques acidentais em ações
